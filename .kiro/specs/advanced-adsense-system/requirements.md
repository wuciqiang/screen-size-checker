# 高级多平台 AdSense 系统需求文档

## 介绍

本文档定义了为 screensizechecker.com 实现高级多平台 AdSense 系统的需求。该系统旨在完全替换现有的广告实现，通过战略性的手动放置、防止布局偏移（CLS）以及为桌面端和移动端提供不同的优化布局来最大化收入。

## 需求

### 需求 1：模块化广告组件系统

**用户故事：** 作为开发者，我希望有可重用的模块化广告组件，以便在整个网站中一致地部署广告。

#### 验收标准

1. WHEN 创建广告组件时 THEN 系统应该提供三种不同类型的广告组件
2. WHEN 使用广告组件时 THEN 每个组件应该包含完整的 AdSense 代码和配置
3. WHEN 部署广告时 THEN 组件应该支持响应式设计和自动格式调整
4. WHEN 集成到构建系统时 THEN 组件应该与现有的 `{{> components/component-name.html }}` 语法兼容

### 需求 2：防止累积布局偏移（CLS）

**用户故事：** 作为用户，我希望页面在加载广告时不会发生布局跳动，以获得流畅的浏览体验。

#### 验收标准

1. WHEN 页面加载时 THEN 每个广告位置应该预留固定的最小高度
2. WHEN 广告加载时 THEN 页面内容不应该发生位置偏移
3. WHEN 广告未能加载时 THEN 预留空间应该保持稳定
4. WHEN 测量 Core Web Vitals 时 THEN CLS 分数应该保持在良好范围内

### 需求 3：响应式布局系统

**用户故事：** 作为用户，我希望在不同设备上看到优化的广告布局，桌面端有侧边栏，移动端有简化布局。

#### 验收标准

1. WHEN 在桌面端（>1440px）浏览时 THEN 系统应该显示三列布局（内容区 + 右侧边栏）
2. WHEN 在移动端（≤1440px）浏览时 THEN 系统应该隐藏右侧边栏，只显示内容区广告
3. WHEN 调整浏览器窗口大小时 THEN 布局应该平滑过渡
4. WHEN 在右侧边栏中时 THEN 广告应该具有粘性定位效果

### 需求 4：战略性广告位置

**用户故事：** 作为网站所有者，我希望在最佳位置放置广告以最大化收入，同时不影响用户体验。

#### 验收标准

1. WHEN 在工具页面时 THEN 系统应该在 H1 标题后和主要工具区域后放置广告
2. WHEN 在博客文章时 THEN 系统应该在标题后、前2-3段后、2/3位置和文章结尾放置广告
3. WHEN 在桌面端时 THEN 系统应该在右侧边栏显示垂直广告
4. WHEN 确定广告位置时 THEN 系统应该考虑内容长度和用户阅读模式

### 需求 5：性能优化

**用户故事：** 作为用户，我希望广告不会显著影响页面加载速度和性能。

#### 验收标准

1. WHEN 页面加载时 THEN 广告脚本应该异步加载
2. WHEN 广告加载失败时 THEN 系统应该有优雅的降级处理
3. WHEN 测量页面性能时 THEN 广告不应该阻塞关键渲染路径
4. WHEN 用户滚动时 THEN 粘性广告应该平滑跟随

### 需求 6：兼容性和集成

**用户故事：** 作为开发者，我希望新的广告系统能够无缝集成到现有的静态网站架构中。

#### 验收标准

1. WHEN 集成到构建系统时 THEN 系统应该与 multilang-builder.js 完全兼容
2. WHEN 使用现有 CSS 时 THEN 系统应该重用现有的样式类而不是创建冲突
3. WHEN 部署时 THEN 系统应该保持纯静态 HTML/CSS/JS 架构
4. WHEN 维护时 THEN 系统应该提供清晰的配置和修改指南
